# Changelog

## Unreleased

No changes yet.

## 0.2.1 - 2026-07-23

- Changed the learnability probe verdict to class-balanced accuracy with per-split raw and balanced
  majority baselines, macro-F1, and a fixed baseline margin, preventing class priors alone from
  producing a shortcut finding.
- Replaced the ambiguous `morphology_dependence` score with conflict gold recall, direct
  shape-hint substitution, other-error, and abstention rates.
- Rejected empty or whitespace-only safety prefixes and reserved domains in both TOML and
  normalized corpus snapshots.
- Enforced annotation-value uniqueness after OCR/spoken transformations and independently during
  corpus validation.
- Made prediction spans fail-closed by default for invalid offsets, labels, duplicates, and
  overlaps, with `--allow-invalid-predictions` as an explicit forensic override.
- Updated the supported-version policy to refer to the latest published release.

## 0.2.0 - 2026-07-23

Byte-determinism break: the generator version is now 2.0.0 and the configuration schema gained
new sections, so 0.2.0 cannot validate or reproduce corpora generated by 0.1.0. Both shipped
example corpora were regenerated.

Audit engine:

- Refactored the audit into a registered check table with graceful degradation, so checks that
  lack their required inputs report `UNMEASURED` instead of failing vacuously.
- Added `shape_entity_shortcut` (P(entity | shape) over identifier-shaped surface forms),
  `intra_split_redundancy` (within-split near-duplicate share plus skeleton ratios),
  `label_marker_shortcuts` (non-cue lexical markers that determine a label), and
  `pervasive_phrase_fingerprint` (placeholder-inclusive 4-grams blanketing the corpus).
- Replaced the SequenceMatcher near-duplicate scan with MinHash/LSH banding plus exact Jaccard
  confirmation (`similarity.py`), removing both a blind spot for differing lead words and the
  quadratic blowup on templated text.
- Replaced `low_value_entropy` with `value_diversity` (per-label distinct-value floors) and added
  the `value_shared_affix` warning that makes dominant value affixes (such as SYN- prefixes)
  visible.
- Added threshold transparency: findings carry measured value, threshold, and threshold source; a
  new `WARN` status; a reference threshold profile (`profiles.py`); a `threshold_strictness`
  finding that flags configurations laxer than the reference; and `audit --profile reference`.
- Made strictness directions explicit for similarity, evidence-support, structural, and probe
  thresholds; affix findings now compare character length with a character ceiling, and
  corpus-level value entropy is informational rather than verdict-bearing.
- Added the opt-in learnability probe (`audit --probe`): deterministic stdlib character-n-gram
  logistic regression measuring kind separability, value-to-label, and context-to-label
  shortcuts empirically.

Generation:

- Persona and organization pools moved into the configuration (`[surfaces]`); split partitioning
  of personas, organizations, identifier letters, and calendar years now interleaves shared pools
  instead of assigning contiguous ranges, avoiding that order-driven cross-split shift.
- Expanded every built-in family from six to thirty authored templates, rotated the constant
  context footer into six variants, and keyed persona/template selection so larger banks are
  actually exercised.
- Near-miss and adjacent hard negatives now generate their values through the configured label
  plugins (with a shared uniqueness pool and an OCR-noised variant), so hard negatives mirror the
  positive value distribution and no shape is entity-exclusive.

New commands and consumers:

- `piicorpus audit-external`: run the config-independent checks, probe, and an informational
  sensitive-content scan on any JSONL, Hugging Face-style JSONL, or CoNLL dataset.
- External audit rejects duplicate input names instead of silently replacing an earlier file.
- `piicorpus score`: score detector span predictions with per-label/family/kind/split
  precision-recall and mechanism diagnostics (cue dependence, morphology dependence,
  over-triggering, noise robustness).
- `piicorpus reproduce`: regenerate from the corpus's own configuration snapshot and
  byte-compare.

Safety and realism:

- Verifier-based safety (`safety.mode = "prefix" | "verifier" | "either"`) with
  `register_value_verifier`, plus realistic-but-reserved value plugins: RFC 2606 emails, NANP
  555-01XX phones, Luhn-invalid card shapes, RFC 5737/3849 documentation IPs, and never-issued
  9XX-XX-XXXX national-id shapes. New `configs/realistic-safe.toml` and `examples/realistic/`.

Extensibility and UX:

- CLI plugin loading via `--plugins` and the `piicorpus.plugins` entry-point group; shape
  signatures are now a registry (`register_shape`) instead of hard-coded demo patterns;
  `register_family` and the other registration hooks are exported from the package root.
- Text reports distinguish warnings, unmeasured checks, and their combined state; callable plugin
  entry points are covered through installed-distribution metadata in the test suite.
- Hugging Face export writes one file per split; every export writes a `labels.json` tag map.
- `validate` is always strict (`--strict` is a deprecated no-op); global `--traceback` re-raises
  errors for debugging; imported unannotated lines are typed `unannotated` (import schema v2) and
  imports are scanned informationally for sensitive-content patterns.
- Fixed a latent crash: text containing lone surrogates now raises `AnnotationError` instead of
  `UnicodeEncodeError`.

## 0.1.0 - 2026-07-22

- Published the source repository, documentation, and first GitHub release.
- Added deterministic generation, independent validation, failure-mode auditing, strict annotated
  text import, and five span-preserving exporters.
- Added fictional demo data, deliberately defective audit cases, documentation, packaging, and CI.
- Removed the label-exclusive hard-negative index marker and replaced it with a class-balanced
  synthetic context/reference suffix.
- Added explicit cue-to-entity links, per-split cue-free checks, and emitted-evidence validation
  for cue/shape contrastive records.
- Made audit and export strict-validation consumers by default, with a failure-preserving forensic
  override for invalid inputs.
- Added content-derived case-ID verification, literal metadata-surface checks, kind-predictive
  marker auditing, and regression corpora for the external audit exploits.
- Extended kind-marker auditing across one-, two-, and three-token features, and made cue-shortcut
  ceilings independently enforceable in every split.
- Added the editable-build helper to the reviewed development lock so fresh no-isolation installs
  follow the documented CI path.
- Pinned the CI runner and reviewed development dependency lock.
- Published release-date citation metadata for the tagged `v0.1.0` release.
- Forced LF checkout semantics for JSON and JSONL corpus files and included the rule in source
  distributions.

No package has been published to a package index.
