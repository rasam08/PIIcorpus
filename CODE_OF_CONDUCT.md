# Code of conduct

## Our standard

Project spaces should be respectful, technically honest, and accessible. Constructive disagreement
is welcome. Harassment, discrimination, threats, deliberate privacy violations, and publication of
another person's private information are not acceptable.

Participants should:

- critique ideas and evidence rather than people;
- disclose uncertainty and correct mistakes openly;
- respect privacy, consent, and licensing boundaries;
- avoid exaggerated safety, accuracy, or compliance claims;
- help newcomers understand both the tooling and its limits.

## Scope and enforcement

This standard applies in project repositories, issue trackers, reviews, and other spaces operated
for the project. Maintainers may edit or remove unacceptable material and may restrict participation
when necessary to protect the community.

Report conduct concerns privately through the repository owner's GitHub profile or, for security
matters, through GitHub private vulnerability reporting. Do not include sensitive personal data in
a public issue.
